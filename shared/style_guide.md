# 统一风格指南

所有子 Agent 必须遵循以下规则。

## 中文排版

### 标点符号（必须使用中文全角）

| 英文符号 | 中文符号 |
|---------|---------|
| `,` | `，` |
| `.` | `。` |
| `:` | `：` |
| `;` | `；` |
| `?` | `？` |
| `!` | `！` |
| `"` | `"` 或 `"` |
| `'` | `'` 或 `'` |

**禁止使用**：
- 英文引号 `""`
- 英文单引号 `''`
- 破折号 `——`

### 标题层级

使用 `#` > `##` > `###`，保持清晰的层级结构。

### 列表

- 可用 `-` 或 `1.`，保持一致
- 嵌套列表使用 4 空格缩进

## 数学公式

### 公式格式

- 使用 display math：`$$...$$`
- 行内公式：`$...$`

### 括号配对（关键规则）

| 用途 | 正确 | 错误 |
|------|------|------|
| 括号 | `\left( ... \right)` | `( ... )` |
| 方括号 | `\left[ ... \right]` | `[ ... ]` |
| 花括号 | `\lbrace ... \rbrace` | `\{ ... \}` |

**注意**：GitHub 渲染时 `\{` 和 `\}` 会出错，必须使用 `\lbrace` 和 `\rbrace`。

### 符号命名

每个公式后必须跟中文解释，所有符号必须定义：

```markdown
$$
\mathcal{L} = -\mathbb{E}_{x \sim p_{\text{data}}} \left[ \log D(x) \right]
$$

其中：
- $\mathcal{L}$：生成器损失函数
- $x$：真实数据样本
- $p_{\text{data}}$：数据分布
- $D(x)$：判别器对 $x$ 的输出
```

## 观点归属

### 必须使用归属标记的场景

- 问题6（学习心得）中的个人见解
- 问题7（应用研究）中的建议
- 任何超出论文原文的分析

### 归属标记格式

```markdown
In my opinion, ...
My analysis is that ...
My read is that ...
```

### 禁止的场景

- 问题1（论文试图解决什么问题）：禁止个人观点
- 问题3（作者提出了什么方法）：禁止个人观点，严格忠实于论文
- 核心术语表：禁止个人观点
- 图片描述：禁止个人观点

## 图片嵌入

### 格式

```markdown
![图1: 描述](./images/figure_1.png)
*图1: 描述*
```

### 规则

- 每张图片必须有中文图注
- 图注格式：`*图 X: 描述*`
- 路径使用相对路径

## 引用规范

### 必须 web_search 的场景

- 延伸阅读推荐
- 外部论文/工具的介绍
- 非论文原文的信息

### 禁止记忆引用

所有外部引用必须来自本次响应中实际运行的 `web_search` 或 `web_fetch`。

### 例外

论文本身引用的工作，且你**完全重复**论文对该工作的描述时，可不搜索。

## 结构规则

### 元信息格式

```markdown
论文标题：（中文译题 / 英文原题）
作者：（第一作者，等）
机构：（如论文有）
发布日期：（YYYY-MM-DD）
来源：arXiv:xxxx / DOI:xxxx
阅读日期：（自动填充）
```

### 核心术语表规则

- 每个术语独占一行
- 术语用**粗体**
- 定义用通俗语言
- 包含英文原文（括号标注）
- 至少 5 个术语

### 一句话总结规则

- 20-50 字
- 不使用"本文"、"该论文"等冗余词
- 格式：`🖊 一句话总结：（内容）`

### FAQ 格式

```markdown
Q: 常见问题？
A: 解答（中文）
```

### 延伸阅读格式

每篇推荐需包含：
- 相关性：为什么相关
- 推荐理由：值得阅读的理由
- 搜索验证：已通过 web_search 验证

## 错误模式警告

### 常见错误

1. **破折号使用**：用逗号、句号、分号或换句代替
2. **引号混用**：只用中文引号 `""`
3. **公式括号**：必须使用 `\left`/`\right` 配对
4. **观点无归属**：问题6和7必须标记观点
5. **图片无图注**：每张图片必须有两个换行分隔的图注
